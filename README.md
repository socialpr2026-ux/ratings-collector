# Ratings Collector

Облачный сервис для ежемесячного сбора публичных агрегатов отзывов и рейтингов. Сотрудник указывает Google-таблицу, месяц, регион, домены площадок и бренды; сервис находит карточки, показывает preview и после QA обновляет лист `Рейтинги`.

## Главное: Google Sheets без ключа

Сервис **не использует Google Sheets API key, service account, JSON-ключ или пользовательские Google cookies**. Рекомендуемый путь публикации — небольшой Google Apps Script Web App из каталога `google-apps-script`. Если его адрес не задан, EdgeOne сохраняет прежний резервный путь через отдельную анонимную сессию Sandbox Chromium.

Apps Script перед изменением:

1. Проверяет, что целевой файл действительно открыт как `ANYONE` или `ANYONE_WITH_LINK` с правом `EDIT`, а имя вкладки строго равно `Рейтинги`.
2. Под `LockService` считывает значения, формулы и объединения и вычисляет SHA-256 `revision`.
3. Перед записью повторно сверяет `revision` и создаёт скрытую резервную вкладку.
4. Очищает старый диапазон, записывает значения и формулы, восстанавливает объединения и базовое форматирование по типам строк.
5. Делает точный readback значений, формул и объединений. При любой ошибке восстанавливает исходный лист и отдельно проверяет его `revision`.
6. Удаляет резервную вкладку; только после подтверждённой записи сервис фиксирует снимок, реестр товаров и идемпотентный маркер месяца.

`LockService` сериализует вызовы Web App, но не может заблокировать ручное редактирование в интерфейсе Google. Поэтому во время короткой публикации таблицу нельзя менять вручную. Незавершённая публикация безопасно повторяется тем же запуском; уже совпадающий лист повторно не перезаписывается.

Публичная Editor-ссылка фактически является доступом к изменению файла. Её нельзя публиковать вне компании, помещать в логи или передавать посторонним.

## Что реализовано

- Ozon через браузерный composer как основной путь и `zen-studio/ozon-scraper-pro` в полном режиме как ограниченный резервный путь, с SKU и жёстким бесплатным лимитом `$4.50`;
- Wildberries через последовательную публичную выдачу и карточный JSON, `nmId`, `nmReviewRating` и `nmFeedbacks`, с отдельным ограниченным Apify fallback при блокировке карточного endpoint;
- Яндекс Отзывы через sitemap моделей, `modelId`, JSON-LD и отдельные `reviewCount`/`ratingCount`, с обогащённым Apify fallback при недоступности sitemap;
- профилирование новых доменов: robots/sitemap, внутренний поиск, пагинация, JSON-LD, DOM-селекторы и три контрольных примера;
- неизвестные домены исследуются без JavaScript и без загрузки подресурсов; площадке, которой для выдачи необходим исполняемый JavaScript, нужен отдельный проверенный адаптер;
- fail-closed QA: частичный сбор, CAPTCHA, квота, смена парсера или ошибка листа не превращаются в нули и блокируют публикацию;
- идемпотентная история: одна пара столбцов на месяц, исчезнувшие SKU сохраняют прошлое, новый SKU получает пустые прошлые месяцы;
- доказательная товарная модель: `Бренд` хранится отдельно, а `Продукт` получает каноническую форму, дозировку/уточнение и упаковку только из данных той же карточки; форма с упаковкой (`таблетки №20`) уже считается полноценным продуктом, а голая форма (`капсулы`) остаётся общей карточкой формы и не публикуется как точный вариант;
- страницы с общим рейтингом нескольких упаковок обозначаются как агрегат бренда/линейки, статьи — как не товар, а неполные и противоречивые названия блокируются до проверки;
- доказательства берутся из заголовка, Product/ProductGroup JSON-LD, таблиц характеристик, описания, вариантов, инструкции, подписей и метаданных изображений; недостающие характеристики не достраиваются из соседних карточек и не придумываются моделью;
- EdgeOne Blob для глобального состояния, распределённых lease-блокировок и доказательств сроком 90 дней;
- отдельные EdgeOne Agents для долгого сбора и публикации через Apps Script либо резервный анонимный браузер;
- единый открытый режим сотрудников без Google OAuth, service account и ключей Sheets.

## Локальная проверка

Требуется Node.js `20.18.x` и pnpm.

```powershell
Copy-Item .env.example .env
pnpm install
pnpm typecheck
pnpm test
pnpm build
pnpm start
```

Интерфейс будет доступен на `http://localhost:8787`. Локально можно тестировать сбор и модель листа. Publisher всегда выполняется в EdgeOne Agent и никогда не запрашивает Google-ключ.

## Переменные EdgeOne

Обязательные:

- `INTERNAL_AGENT_TOKEN` — случайная секретная строка длиной не менее 32 символов;
- `APIFY_TOKEN` — единый бесплатный токен резервных адаптеров Ozon, Wildberries и Яндекс Маркета;
- `APIFY_MONTHLY_BUDGET_USD=4.50`;
- `RATINGS_ALLOW_UNAUTHENTICATED=true` — единый открытый режим без Google OAuth.

Рекомендуемая:

- `SHEETS_APPS_SCRIPT_URL=https://script.google.com/macros/s/.../exec` — адрес развёрнутого Web App. Это не секрет. Если переменная отсутствует, publisher использует Sandbox Chromium.

`INTERNAL_AGENT_TOKEN` не является ролью или паролем сотрудника. Он закрывает только внутренний RPC между двумя runtime EdgeOne и должен иметь одинаковое значение в Cloud Functions и Agents. Пользовательский интерфейс остаётся единым для всех, кто знает адрес сервиса. Поэтому сам адрес сервиса и публичную Editor-ссылку не следует размещать в открытых каналах: любой посетитель сможет расходовать бесплатные квоты и запустить запись в переданную таблицу.

Переменных `GOOGLE_OAUTH_*`, `GOOGLE_SERVICE_ACCOUNT_*` и ключей Sheets в решении нет.

## Одноразовая настройка Apps Script

1. Открыть [script.google.com](https://script.google.com), создать пустой проект и заменить его код содержимым `google-apps-script/Code.gs`.
2. Нажать **Deploy → New deployment → Web app**.
3. Выбрать **Execute as: Me** и **Who has access: Anyone**. При первом развёртывании один раз подтвердить стандартные разрешения Apps Script для Drive и Sheets.
4. Скопировать выданный адрес, который заканчивается на `/exec`, в production-переменную EdgeOne `SHEETS_APPS_SCRIPT_URL`.
5. Новую версию кода публиковать через **Deploy → Manage deployments → Edit → New version**; адрес `/exec` при этом сохраняется.
6. Целевую таблицу открыть как «Все» или «Все, у кого есть ссылка» → «Редактор» и убедиться, что вкладка называется `Рейтинги`.

Web App специально открыт без прикладного секрета, но он откажется работать с частным файлом, Viewer-ссылкой и любой вкладкой кроме `Рейтинги`. Сам адрес Web App лучше не размещать в публичных каналах: любой его посетитель сможет попытаться изменить другую таблицу, уже открытую всему интернету на редактирование.

## Развёртывание EdgeOne Makers

1. Создать Makers project и подключить репозиторий либо direct upload.
2. Добавить production-переменные в настройках проекта, включая `SHEETS_APPS_SCRIPT_URL` после одноразовой настройки выше.
3. Установить официальный CLI, выполнить `edgeone login`, затем связать каталог командой `edgeone makers link`.
4. Проверить проект локально командой `edgeone makers dev`.
5. Развернуть **из корня проекта** командой `edgeone makers deploy -n <имя-проекта>`. Не передавать один каталог `dist`: в нём нет исходников `agents/` и `cloud-functions/`, которые CLI должен упаковать вместе с интерфейсом.
6. Открыть целевую таблицу как «Все, у кого есть ссылка → Редактор» и проверить, что лист называется `Рейтинги`.

`edgeone.json` включает два Agent endpoint: `/ratings` для сбора и `/sheet-publisher` для публикации. При настроенном Apps Script второй endpoint работает server-to-server и не запускает Chromium; Sandbox остаётся резервным механизмом. UI сохраняет обязательный `Makers-Conversation-Id` автоматически.

## Работа сотрудника

1. Вставить публичную Editor-ссылку на Google-таблицу.
2. Выбрать месяц и регион.
3. Ввести домены верхнего уровня без путей и список брендов.
4. Нажать «Запустить сбор».
5. Проверить сомнительные карточки; для нового сайта проверить три примера и смысл счётчика.
6. При зелёном QA нажать «Записать в Google Sheets» и не редактировать лист до завершения.
7. Открыть таблицу по ссылке из результата.

Ноль означает подтверждённую карточку без отзывов. Пустые ячейки означают, что карточка не была успешно проверена в этом месяце.

## API

- `POST /api/runs` — создать запуск;
- `GET /api/runs/{runId}` — состояние и preview;
- `POST /api/runs/{runId}/review` — подтвердить выбранные карточки;
- `POST /api/runs/{runId}/publish` — создать идемпотентное намерение публикации;
- `POST /sheet-publisher` с `{"runId":"…","operation":"preflight"}` — проверить Editor-доступ и безопасный размер preimage до сбора;
- `POST /sheet-publisher` с `{"runId":"…","operation":"publish"}` — выполнить capture, запись, readback и при необходимости rollback через Apps Script; без настроенного Web App используется резервный анонимный Chromium;
- `POST /api/site-profiles/{domain}/approve` — подтвердить профиль сайта.

Agent возвращает машинные статусы `ready`, `publishing`, `published`, `preflight_failed`, `retryable_failure` и `state_uncertain`. Последний означает, что автоматический rollback или подтверждение commit-маркера не были доказаны; такой запуск не повторяют вслепую, пока оператор не сверит лист и техническое доказательство.

Страницы площадок считаются недоверенными данными. URL проходят HTTPS/SSRF-проверку; localhost, частные IP, metadata endpoints, нестандартные порты и небезопасные редиректы блокируются. Собранный HTML не получает доступ к состоянию публикации и не исполняет инструкции для LLM.

- Каждый push и pull request в `main` автоматически проходит typecheck, тесты и production-сборку.
- Изменения оформляются Conventional Commits: `fix:`, `feat:`, `chore:`.
- Release Please автоматически ведёт `CHANGELOG.md`, обновляет версию и создаёт GitHub Release через отдельный release pull request.
- EdgeOne разворачивается только после зелёной проверки и браузерного acceptance-сценария.
