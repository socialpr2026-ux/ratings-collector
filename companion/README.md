# Локальный сборщик Ozon

## Путь сотрудника

В веб-интерфейсе нажмите «Скачать помощник», откройте полученный файл `ratings-ozon-helper.cmd` и оставьте его окно открытым. Затем нажмите «Проверить и собрать»: помощник откроет отдельный Chrome и вернёт в текущий запуск только SKU, название, отзывы и рейтинг.

Браузер не имеет права сам запускать программы Windows, поэтому первая установка всегда подтверждается открытием скачанного файла. После запуска интерфейс проверяет `http://127.0.0.1:8765/health`; молчаливого обещания автостарта нет. Установка сохраняется в `%LOCALAPPDATA%\RatingsCollector\app`, а профиль Chrome — в `%LOCALAPPDATA%\RatingsCollector\ozon-profile`.

Локальный companion нужен только когда Ozon блокирует облачный адрес. Он запускает установленный Chrome с отдельным профилем и запрашивает публичную выдачу через обычное подключение сотрудника. В облачный сервис возвращаются только SKU, название, число отзывов и рейтинг.

## Почему выбран этот вариант

| Вариант | Установка на Windows | Обычный IP сотрудника | Поддержка | Решение |
| --- | --- | --- | --- | --- |
| Node + установленный Chrome | один командный файл | да | один небольшой процесс | выбран |
| Docker + Chromium | Docker Desktop, образ, больше памяти | да | сложнее обновлять и объяснять | лишняя сложность |
| Chrome extension | ручная установка или Chrome Web Store | да | подпись, разрешения и версии Manifest | возможен позднее, но не для v1 |
| Bookmarklet | формально без установки | да | хрупок, ручной запуск на каждой странице | не подходит для полного сбора |

## Запуск

1. Установить Google Chrome и Node.js 20 или новее.
2. Дважды нажать `Установить и запустить.cmd`.
3. Оставить открывшееся окно свёрнутым и запустить сбор в основном веб-интерфейсе.
4. Если Chrome показывает проверку Ozon, пройти её один раз и нажать «Повторить сбор» в веб-интерфейсе. Для companion используется отдельный профиль в `%LOCALAPPDATA%\RatingsCollector\ozon-profile`.

Chrome запускается как обычное пользовательское приложение, а companion подключается к нему через локальный DevTools-порт. В этом режиме `navigator.webdriver` остаётся выключенным; сам companion не решает CAPTCHA и не скрывает автоматизацию от площадки.

Сервис слушает только `127.0.0.1:8765`, принимает запросы только от production-интерфейса и локальной разработки, а его единственная рабочая команда принимает список брендов. Произвольный URL передать нельзя.

## Контракт для веб-интерфейса

- Проверка доступности: `GET http://127.0.0.1:8765/health`.
- Сбор: `POST http://127.0.0.1:8765/v1/ozon/discover` с JSON `{ "brands": ["Кагоцел"], "region": "Москва" }`.
- Ответ: `{ "version": 1, "observations": [{ "listingId", "brand", "canonicalUrl", "product", "reviews", "rating", "status", "capturedAt" }], "partitions": [{ "brand", "status", "discovered", "collected" }] }`.

Основной UI проверяет companion только после честной блокировки облачного Ozon, получает короткую одноразовую сессию и импортирует результат в существующий запуск. Companion сам не получает Google-доступ и не публикует таблицу.
